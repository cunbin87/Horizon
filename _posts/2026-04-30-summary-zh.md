## [Mozilla 反对 Chrome 的 Prompt API，因模型耦合问题](https://github.com/mozilla/standards-positions/issues/1213) ⭐️ 8.0/10

Mozilla 正式反对 Chrome 的 Prompt API，理由是提示词与特定语言模型紧密耦合，缺乏模型中立性。该反对意见发布在 Mozilla 的 standards-positions GitHub 仓库中。 这一反对意见凸显了对浏览器垄断以及 AI 集成到浏览器中的担忧，可能影响网络标准的制定和用户隐私。它可能影响未来 AI 功能在不同浏览器中的设计和采用方式。 由 Google 提出的 Prompt API 在 Chrome 中使用 Gemini Nano 模型，并要求开发者承认 Google 的生成式 AI 禁止使用政策。Mozilla 认为该 API 不具有模型中立性，将提示词绑定到单一模型，违反了网络标准原则。

hackernews · jaffathecake · Apr 30, 07:43

**背景**: Prompt API 是 Web 机器学习社区的一项提案，允许 Web 开发者直接从浏览器中调用语言模型。Mozilla 的反对基于网络 API 应独立于模型的原则，以避免供应商锁定并确保中立性，类似于将网络中立性概念应用于浏览器。

## [Copy Fail：732 字节即可获取所有主流 Linux 发行版 root 权限](https://copy.fail/) ⭐️ 8.0/10

Linux 内核 AF_ALG 加密接口中存在一个严重漏洞（CVE-2026-31431），非特权用户仅需 732 字节的利用代码即可获取 root 权限。该漏洞名为“Copy Fail”，影响所有主流 Linux 发行版。 该漏洞暴露了巨大的攻击面，因为 AF_ALG 默认启用且非特权用户可访问。供应商低估了其严重性，导致许多系统未打补丁，面临完全被攻陷的风险。 该漏洞位于内核加密子系统的 authencesn 实现中，通过精确的套接字选项和辅助数据操作触发。利用代码使用 splice()操作页缓存，无需竞争条件即可实现权限提升。

hackernews · unsnap_biceps · Apr 29, 18:13

**背景**: AF_ALG 是 Linux 内核套接字接口，允许用户空间程序访问内核加密操作。它多年前未经充分审查就被加入，被认为过于复杂，暴露了巨大的攻击面。大多数用户空间应用程序可以使用自己的加密库，因此 AF_ALG 在很大程度上是不必要的。

## [GCC 16 发布，支持 C++26 反射与显式生命周期管理](https://gcc.gnu.org/gcc-16/changes.html) ⭐️ 8.0/10

GCC 16.1 作为 GCC 16 的首个稳定版本，引入了对 C++26 反射（P2996）和显式生命周期管理（P2590R2）的支持，通过 std::start_lifetime_as 实现编译时类型内省和安全的类型双关。 这些特性显著提升了 C++ 编写更安全、更具表现力代码的能力，尤其适用于零拷贝反序列化和泛型编程，并可能加速 C++26 在生产环境中的采用。 C++26 反射使用 ^ 运算符和 std::meta 命名空间进行编译时内省，而显式生命周期管理提供 std::start_lifetime_as，可在不引发未定义行为的情况下合法地将内存重新解释为给定类型。

hackernews · HeliumHydride · Apr 30, 11:40

**背景**: 反射允许程序在编译时检查和操作自身结构，从而实现自动序列化等功能。显式生命周期管理解决了 C++ 中长期存在的问题：通过 reinterpret_cast 进行类型双通常常导致未定义行为；std::start_lifetime_as 提供了一种符合标准的替代方案。
